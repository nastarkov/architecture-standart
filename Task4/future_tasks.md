# Крупные задачи для планирования по системам из ADR

## 1. Сайт банка

### 1.1. Отображение депозитных продуктов
- Реализовать витрину депозитных продуктов и ставок для новых клиентов
- Обеспечить получение списка депозитов из Deposit Application Service
- Реализовать отображение параметров продукта: срок, валюта, ставка, условия открытия

### 1.2. Оформление заявки новым клиентом
- Реализовать форму заявки на открытие депозита
- Организовать валидацию вводимых данных (ФИО, телефон и др.)
- Реализовать отправку заявки в Deposit Application Service

---

## 2. Интернет-банк

### 2.1. Получение и отображение ставок
- Реализовать запрос актуальных ставок и депозитных продуктов из Deposit Application Service
- Отобразить доступные и персонализированные ставки для существующего клиента
- Поддержать выбор параметров депозита: счёт списания, сумма, срок и др.

### 2.2. Подача заявки на депозит
- Реализовать сценарий создания заявки на депозит
- Обеспечить передачу параметров заявки в Deposit Application Service
- Реализовать получение статуса обработки заявки

### 2.3. Подтверждение операции
- Реализовать сценарий подтверждения операции через SMS-код
- Обеспечить взаимодействие с Notification Service через Deposit Application Service
- Обработать ошибки неверного или просроченного кода

---

## 3. Система кол-центра

### 3.1. Приём и обработка заявок новых клиентов
- Обеспечить получение заявок новых клиентов из Deposit Application Service
- Реализовать отображение карточки заявки для менеджера кол-центра
- Поддержать изменение статуса обработки заявки менеджером

### 3.2. Поддержка консультационного сценария по ставкам
- Реализовать раздел просмотра актуальных депозитных ставок для сотрудников кол-центра
- Поддержать фильтрацию ставок по основным параметрам продукта
- Обеспечить обновление данных ставок из новой депозитной системы

### 3.3. Интеграция и аудит действий
- Реализовать журнал получения и обработки заявок
- Обеспечить аудит действий операторов
- Реализовать обработку ошибок интеграции с Deposit Application Service

---

## 4. Deposit Application Service

### 4.1. Управление сценариями открытия депозита
- Реализовать orchestration для сценария нового клиента
- Реализовать orchestration для сценария существующего клиента
- Разделить обработку сценариев сайта, интернет-банка и кол-центра

### 4.2. Управление заявками
- Реализовать создание, хранение и изменение статусов заявок
- Поддержать жизненный цикл заявки от создания до завершения
- Обеспечить хранение истории обработки заявки

### 4.3. Работа с депозитными продуктами и ставками
- Реализовать получение и предоставление актуальных депозитных продуктов и ставок
- Поддержать выдачу ставок для сайта, интернет-банка и кол-центра
- Обеспечить единый прикладной API для чтения продуктовой информации

### 4.4. Интеграция с внутренними и внешними системами
- Реализовать взаимодействие с ABS ACL для запросов и команд в АБС
- Реализовать передачу заявок в систему кол-центра
- Реализовать взаимодействие с Notification Service

### 4.5. Надёжность и эксплуатация
- Реализовать логирование, трассировку и мониторинг бизнес-операций
- Обеспечить обработку ошибок и повторные попытки там, где это допустимо
- Реализовать технические механизмы для SLA и поддержки эксплуатации

---

## 5. Notification Service

### 5.1. Отправка SMS-кодов подтверждения
- Реализовать отправку одноразовых SMS-кодов для подтверждения операций
- Поддержать проверку корректности параметров отправки
- Обеспечить журналирование факта отправки

### 5.2. Отправка SMS-уведомлений
- Реализовать отправку уведомлений по статусам депозитной заявки
- Поддержать шаблоны сообщений для разных сценариев
- Обеспечить контроль ошибок доставки

### 5.3. Интеграция с SMS-шлюзом
- Реализовать адаптер интеграции с SMS Gateway
- Обработать ответы и ошибки внешнего шлюза
- Обеспечить технический мониторинг канала отправки

---

## 6. ABS ACL

### 6.1. Изоляция новой системы от модели АБС
- Реализовать слой адаптации между Deposit Application Service и интерфейсами АБС
- Выполнить преобразование запросов и ответов в формат доменной модели новой системы
- Скрыть особенности legacy-интерфейсов АБС от новой депозитной системы

### 6.2. Операции по депозитам
- Реализовать вызовы АБС для подтверждения условий депозита
- Реализовать вызовы АБС для открытия депозитного счёта
- Реализовать получение необходимых данных по ставкам и депозитным продуктам

### 6.3. Устойчивость интеграции
- Реализовать обработку ошибок и таймаутов вызовов АБС
- Обеспечить журналирование интеграционных вызовов
- Подготовить механизмы деградации или отказобезопасного поведения при проблемах с АБС

---

## 7. АБС

### 7.1. Подтверждение и учёт депозитных операций
- Обеспечить обработку команд подтверждения условий депозита
- Обеспечить открытие депозитного счёта по заявке
- Поддержать завершение сценария для новых и существующих клиентов

### 7.2. Работа со ставками
- Обеспечить ведение и актуализацию ставок в рамках текущего процесса
- Поддержать предоставление данных по ставкам для новой депозитной системы
- Сохранить операционный процесс работы сотрудников бэк-офиса со ставками

### 7.3. Интеграционная готовность
- Подготовить интерфейсы для вызова через ABS ACL
- Определить ограничения по нагрузке, таймаутам и окнам доступности
- Обеспечить контроль корректности входящих интеграционных вызовов

---

## 8. SMS-шлюз

### 8.1. Поддержка отправки SMS
- Обеспечить приём запросов на отправку SMS-кодов и уведомлений
- Поддержать требуемую производительность и надёжность канала
- Обеспечить возврат статусов доставки или ошибок

### 8.2. Эксплуатация интеграции
- Подготовить мониторинг доступности шлюза
- Обеспечить журналирование технических ошибок
- Согласовать лимиты, правила ретраев и обработку недоставленных сообщений
